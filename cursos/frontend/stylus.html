Introducción

Stylus es un pre-procesador de css. Lo que quiere decir que este código se copila para luego guardarlo en un archivo CSS. Por lo tanto, se traduce del lenguaje de Stylus al lenguaje de CSS.

Nota:
Stylus no usa Ruby como backend sino NodeJS.

Nota:
Se recomienda no guardar los estilos en la raíz. Crear una carpeta llamada stylus, donde estaran los archivos stylus y otra carpeta css para los archivos finales.

Resumen:

Un preprocesador nos permite compilar código escrito en stylus y convertirlo a css que entienda el navegador. Cuando modificamos un archivo de stylus no necesariamente debemos ir a compilar eso, podemos hacer que algo automáticamente detecte el cambio y lo compile, podemos usar una ui en vez de la consola, se puede usar codekit o prepos. Un archivo de stylus tiene la extensión .styl.

Pasos básicos para crear un proyecto con Stylus

  • Se crean las carpetas del proyecto(images, stylus, css, ..)
  • En un editor de código crear un archivo stylus(styles.styl).
  • Se arrastran o importan las carpetas creadas a la UI que compila el código stylus.
  • Al editar el archivo .styl desde un editor de texto automáticamente la UI genera el css correspondiente.

Sintaxis

Datos importantes:

  • Stylus es un preprocesador que difiere un poco del resto por su sintaxis.
  • Su sintaxis es anidada.
  • En Stylus no es necesario usar los brackets y ; pero debemos recordar que para tener mejores prácticas deberíamos usarlas.

Stylus no requiere el uso de las llaves o brackets {...} 

              /*En css*/
              body{
                background: red;
              }

              /*En stylus*/
              body
                background: red;

punto y coma. 

              /*En css*/
              body{
                background: red;
                color: white;
              }

              /*En stylus*/
              body
                background: red
                color: white

Anidación

Gracias a las anidaciones podemos añadir propiedades a través de la sintaxis de Stylus. Además, esto nos permite referenciar al padre.

Sub-anidaciones hacemos referencia a todos los elementos que estén dentro de las etiquetas padres. 

              body, .light-theme
                background: lightyellow
                color: white
                .primer-hijo
                  margin-bottom: 24px

Child con el uso de "">"" hacemos referencia a los hijos directos del elemento padre. 

              body, .light-theme
                background: lightyellow
                color: white
                > .primer-hijo
                  margin-bottom: 24px

Referencia al padre con el uso de ""&"" hacemos referencia al nombre del elemento padre. 

              .psudoelement
                width: 300px
                &:before
                  content: 'hola'
               .ie-8 &
                &:before
                content: 'ciao'

Nota:
No adinar más de tres veces.

Ejemplo de modulación. 

              .btn
                background: white
                color: black
                border-radius: 3px
                &-alert
                  background: red
                  color: white
                &-warn
                  background: orange
                  color: white

Variables

Una de las mayores ventajas que tienen los preprocesadores son las variables, estas nos permiten controlar un entorno porque podemos manipular un entorno. Esto quiere decir que podemos cambiar muchas cosas a través de muchas variables.

Controlar un entorno por medio de pequeños elementos, es más cómodo reutilizar variables, luego con un pequeño cambio ajustar ese valor.

            //Variables
            font-size-normal = 18px
            font-size-small = font-size-normal - 2px
            font-size-paragraph = font-size-normal

            //Text
            p
              font-size: font-size-paragraph

            span
              font-size: font-size-normal
              &.small
                font-size: font-size-small

Estructura css

Una de las ventajas de los preprocesadores es que podemos crear diferentes parciales para organizar nuestro código. Cuando creamos un parcial debemos tener un _, lo que quiere decir que debe ser importado en un archivo principal.

Los archivos que tiene un _ no se compila. Es algo que vamos a usar en otro archivo Para importar archivos usamos @import "_nombre_archivo". 

            @import "_variables.styl"

Nota:
Recomendado tener un solo archivo .styl que importa los demás archivos con esa extensión, y ese solo archivo es el que se compila y genera el css de nuestro sitio. Al importar todo se va a imprimir en el orden en que se esta importando, tener en cuenta la cascada.

Mixins

Los mixins son declarados a través de nombres. Es importante que siempre añadamos el prefijo Mixin. Un mixin nos permite reutilizar valores dentro del mismo CSS Se declara facilmente con su nombre, y como buena práctica usamos minix- antes del nombre que queremos darle. 

            mixin-max-width()
              max-width: 1024px
              margin-left: auto
              margin-right: auto

            /*Uso del mixin*/
            .container
              mixin-max-width()

Nota:
Un mixins no compila, no nombrar a los mixins con nombres ya reservados de css.

Mixins Paramétricos

¿Cómo solucionamos que un diseñador quiera meter más información dentro de un solo mixin? No debemos hacer otro mixin, sino que podemos usar los mixin paramétricos. Aquí lo que debemos incluir son parámetros para que podamos guardar cierta información que es nueva dentro del mixin.

Para dar parámetros a un mixin lo usamos dentro del paréntesis:

            mixin-max-width(width)
              max-width: width
              margin-left: auto
              margin-right: auto

            /*Uso del mixin*/
            .container
              mixin-max-width(800px)

Para usar valores por defecto igualamos el parametro al valor que queremos:

            page-max-width: 1024px /*Variable*/
            mixin-max-width(width = page-max-width)
              max-width: width
              margin-left: auto
              margin-right: auto

            /*Uso del mixin*/
            .container
              mixin-max-width(800px)
              /*Si no se le agrega valor en el parametro toma por defecto el de la variable page-max-width*/

Más ejemplos

            page-max-width: 1024px /*Declaramos una variable en nuestra sección de variables de proyecto*/

            /*Luego agregamos esa variable como valor por defecto del mixins*/
            /*El mixin debe estar declarado en nuestro archivo de mixins*/
            mixin-max-width(width = page-max-width)
            max-width: width
            margin-left: auto
            margin-right: auto
            
            /*Uso del mixin*/
            
            /*Aplicando el mixin con valor custom*/
            .container
            mixin-max-width(800px)
            
            /*Aplicando el mixin con valor custom*/
            .section
            mixin-max-width(90%)
            margin-bottom 1em
            
            /*Aplicando el mixin con valor por defecto*/
            .section-landing
              mixin-max-width()

Compilación en css:

            .container{
              max-width: 800px;
              margin-left: auto;
              margin-right: auto;
            }

            .section{
              max-width: 90%;
              margin-left: auto;
              margin-right: auto;
              margin-bottom 1em;
            }

            .section-landing{
              max-width: 1024px;
              margin-left: auto;
              margin-right: auto;
            }

Funciones internas

En stylus podemos asignar variables dinámicas y asignarles funciones matemáticas que ejecutaras cada que sean llamadas.
Por ejemplo, podremos utilizar la declaracion: 

            padding : mixin-padding-small(2px)

Ubicada en la carpeta site.styl
y pasarle la función:

              mixin-padding-small(a)
                a + minipadding

Donde minipadding proviene de la carpeta _variables.styl y equivale a :

              minipadding = padding / 2

Condicionales

Stylus cuenta con condicionales if que vienen muy útiles a la hora de asegurarnos que algo tiene un valor y que se ejecute algo si eso se cumple.

            tone(dark = true)
              if dark
                text-color: white
                background-color: black
              else
                text-color: black
                background-color: white

            /*Aplicación*/
            .container
              tone(dark)

            /*En CSS, si la condición se cumple*/
            .container{
              text-color: white;
              background-color: black;
            }

Para que la condición no se cumpla se coloca en el parametro false.

            .container
              tone(false)

            /*En CSS, la condición no se cumple*/
            .container{
              text-color: black;
              background-color: white;
            }

Introducción

Stylus es un pre-procesador de css. Lo que quiere decir que este código se copila para luego guardarlo en un archivo CSS. Por lo tanto, se traduce del lenguaje de Stylus al lenguaje de CSS.

Nota:
Stylus no usa Ruby como backend sino NodeJS.

Nota:
Se recomienda no guardar los estilos en la raíz. Crear una carpeta llamada stylus, donde estaran los archivos stylus y otra carpeta css para los archivos finales.

Resumen:

Un preprocesador nos permite compilar código escrito en stylus y convertirlo a css que entienda el navegador. Cuando modificamos un archivo de stylus no necesariamente debemos ir a compilar eso, podemos hacer que algo automáticamente detecte el cambio y lo compile, podemos usar una ui en vez de la consola, se puede usar codekit o prepos. Un archivo de stylus tiene la extensión .styl.

Pasos básicos para crear un proyecto con Stylus

  • Se crean las carpetas del proyecto(images, stylus, css, ..)
  • En un editor de código crear un archivo stylus(styles.styl).
  • Se arrastran o importan las carpetas creadas a la UI que compila el código stylus.
  • Al editar el archivo .styl desde un editor de texto automáticamente la UI genera el css correspondiente.

Sintaxis

Datos importantes:

  • Stylus es un preprocesador que difiere un poco del resto por su sintaxis.
  • Su sintaxis es anidada.
  • En Stylus no es necesario usar los brackets y ; pero debemos recordar que para tener mejores prácticas deberíamos usarlas.

Stylus no requiere el uso de las llaves o brackets {...} 

              /*En css*/
              body{
                background: red;
              }

              /*En stylus*/
              body
                background: red;

punto y coma. 

              /*En css*/
              body{
                background: red;
                color: white;
              }

              /*En stylus*/
              body
                background: red
                color: white

Anidación

Gracias a las anidaciones podemos añadir propiedades a través de la sintaxis de Stylus. Además, esto nos permite referenciar al padre.

Sub-anidaciones hacemos referencia a todos los elementos que estén dentro de las etiquetas padres. 

              body, .light-theme
                background: lightyellow
                color: white
                .primer-hijo
                  margin-bottom: 24px

Child con el uso de "">"" hacemos referencia a los hijos directos del elemento padre. 

              body, .light-theme
                background: lightyellow
                color: white
                > .primer-hijo
                  margin-bottom: 24px

Referencia al padre con el uso de ""&"" hacemos referencia al nombre del elemento padre. 

              .psudoelement
                width: 300px
                &:before
                  content: 'hola'
               .ie-8 &
                &:before
                content: 'ciao'

Nota:
No adinar más de tres veces.

Ejemplo de modulación. 

              .btn
                background: white
                color: black
                border-radius: 3px
                &-alert
                  background: red
                  color: white
                &-warn
                  background: orange
                  color: white

Variables

Una de las mayores ventajas que tienen los preprocesadores son las variables, estas nos permiten controlar un entorno porque podemos manipular un entorno. Esto quiere decir que podemos cambiar muchas cosas a través de muchas variables.

Controlar un entorno por medio de pequeños elementos, es más cómodo reutilizar variables, luego con un pequeño cambio ajustar ese valor.

            //Variables
            font-size-normal = 18px
            font-size-small = font-size-normal - 2px
            font-size-paragraph = font-size-normal

            //Text
            p
              font-size: font-size-paragraph

            span
              font-size: font-size-normal
              &.small
                font-size: font-size-small

Estructura css

Una de las ventajas de los preprocesadores es que podemos crear diferentes parciales para organizar nuestro código. Cuando creamos un parcial debemos tener un _, lo que quiere decir que debe ser importado en un archivo principal.

Los archivos que tiene un _ no se compila. Es algo que vamos a usar en otro archivo Para importar archivos usamos @import "_nombre_archivo". 

            @import "_variables.styl"

Nota:
Recomendado tener un solo archivo .styl que importa los demás archivos con esa extensión, y ese solo archivo es el que se compila y genera el css de nuestro sitio. Al importar todo se va a imprimir en el orden en que se esta importando, tener en cuenta la cascada.

Transformaciones de sesgados

Sesgar es esa inclinación que puede sufrir un elemento gracias a una transformación en CSS.
Se puede sesgar un elemento tanto de un solo eje, como de ambos:

  • skew: Sesgar en el eje X. 
                    transform: skew(20deg);
  • skew(valorX, valorY): Sesgar en el eje X e Y. 
                    transform: skew(10deg, 45deg);
  • Sesgar en X e Y, de manera independiente: 
                    transform: skewX(10deg);
                transform: skewY(45deg);

Nota:
Esta propiedad recibe valores en grados (deg) y radianes (rad).

Punto de transformación

El punto de transformación de un elemento está clavado en alguna parte de la interfaz, y por defecto se encuentra en el centro.

Sin embargo, podemos jugar con la posición de este punto según el tipo de animación que queramos crear.

            transform: rotate(45deg);
            transform-origin: bottom left;

Animaciones

Configurando animación: Se utiliza la palabra clave @keyframes seguido del nombre que queramos darle a la animación, dentro de las llaves se especifica lo que hace dicha animación y esto se hace con estados:

            /* @keyframes nombreAnimación */
            @keyframes cuadrado{
              /* Estado inicial de animación */
              0%{
                opacity:1;
              }
              /* Estado final de animación */
              100%{
                opacity:0;
              }
            }

Para crear una animación CSS usaremos la propiedad animation y sus sub-propiedades, con las cuales podemos configurar el ritmo, la duración, etc.

Las propiedades de animation son:

  • animation-name: Especifica el nombre de la regla @keyframes que describe los fotogramas de la animación. 
                    animation-name: rebote;
  • animation-duration: Indica la cantidad de tiempo que la animación consume en completar su ciclo (duración). 
                    animation-duration: 1s;
  • animation-delay: Tiempo de retardo entre el momento en que el elemento se carga y el comienzo de la secuencia de la animación. 
                    animation-delay: 1s;
  • animation-direction: Indica si la animación debe retroceder hasta el fotograma de inicio al finalizar la secuencia o si debe comenzar desde el principio al llegar al final. 
                    animation-direction: reverse;/*Empieza desde el último estado 100%.*/
                animation-direction: alternate; /*La animación al terminar empieza nuevamente pero desde el último estado en vez del inicio.*/
  • animation-iteration-count: El número de veces que se repite. Podemos indicar infinite para repetir la animación indefinidamente. 
                    /*Cuantas veces va a ocurrir la animación*/
                animation-iteration-count: 1;
                animation-iteration-count: infinite;
  • animation-play-state: Permite pausar y reanudar la secuencia de la animación. 
                    animation-play-state: running; /*Valor por defecto*/
                animation-play-state: paused;
  • animation-timing-function: Indica el ritmo de la animación, es decir, como se muestran los fotogramas de la animación, estableciendo curvas de aceleración. 
                    animation-timing-function: ease; /*Valor por defecto*/
                animation-timing-function: ease-in; 
                animation-timing-function: ease-in-out; 
                animation-timing-function: linear; 
                animation-timing-function: cubic-bezier(1, 1, 1, 1);
  • animation-fill-mode: Especifica qué valores tendrán las propiedades después de finalizar la animación (los de antes de ejecutarla, los del último fotograma de la animación o ambos). 
                    animation-fill-mode: forwards;

Aceleración y curva de bezier

Aceleración y movimientos curvos a los elementos web por medio de animation-timing-function.

Valores de animation-timing-function:

  • ease: Por defecto.
  • ease-in: Se suaviza al entrar o al inicio.
  • ease-out: Se suaviza al salir o al finalizar.
  • ease-in-out: Se suaviza al entrar y salir, básicamente se acelera solo la mitad de la animación.
  • steps(frames por segundo): como el lag en los juegos en linea, mientras más bajo será menos fluido, entre más fps más suave la animación.

Múltiples animaciones

Para nombrar más de una animación podemos nombrarlas en la misma línea de código:

            animation-name: nombre1, nombre2;

Animaciones CSS desde JS

Para detectar eventos de animaciones a través de Javascript se pueden usar los siguientes:

  • animationstart: Para ver si la animación inicio.
  • animationend: Para ver si la animación termino.

Tambien se puede crear animaciones desde Javascript, ejemplo:

            $cuadrado.style.animationName = 'cuadrado';
            $cuadrado.style.animationDuration = '1s';

Para crear animaciones desde JavaScript se utiliza element.animate, que es parte de la especificación del API de animaciones de JavaScript.

            // element.animate(keyframes = [], option = {})
            $pelota.animate([
              // from
              transform: 'translateX(0)'
            },
            // to
            {
              transform: 'translateX(500px)'
            }  
            ], {
              duration: 1000,
              delay: 1000,
              direction: 'normal',
              easing: 'linear',
              iterations: Infinite,
              fill: 'forwards'
            })
